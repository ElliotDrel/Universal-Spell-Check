```python
LOGS_DIR = Path(__file__).resolve().parent / "logs"
MAX_WEEKLY_LOG_SIZE = 5 * 1024 * 1024

def get_week_start_stamp(dt):
    """Return the Monday-based week start stamp for a datetime."""
    return (dt - timedelta(days=dt.weekday())).strftime("%Y-%m-%d")

def get_week_end_stamp(dt):
    """Return the Sunday-based week end stamp for a datetime."""
    return (dt + timedelta(days=(6 - dt.weekday()))).strftime("%Y-%m-%d")

def build_weekly_log_path(week_start_stamp, week_end_stamp, suffix_index=0):
    suffix = f"-{suffix_index + 1}" if suffix_index > 0 else ""
    return LOGS_DIR / f"spellcheck-{week_start_stamp}-to-{week_end_stamp}{suffix}.jsonl"

def resolve_weekly_log_path(week_start_stamp, week_end_stamp, pending_bytes):
    suffix_index = 0
    while True:
        path = build_weekly_log_path(week_start_stamp, week_end_stamp, suffix_index)
        if not path.exists():
            return path
        if path.stat().st_size + pending_bytes <= MAX_WEEKLY_LOG_SIZE:
            return path
        suffix_index += 1
```

**Architecture:**
- Duplicate the week-routing functions from `generate_log_viewer.py` (get_week_start_stamp, get_week_end_stamp, build_weekly_log_path, resolve_weekly_log_path) so the script is self-contained.
- Use SCRIPT_DIR / "logs" as LOGS_DIR, MAX_WEEKLY_LOG_SIZE = 5 * 1024 * 1024.

**Format 1 Parser** (pipe-delimited, `logs/OLD - spellcheck.log`):
- Read with `encoding="utf-8", errors="replace"` (file has non-UTF-8 bytes from Windows clipboard).
- **Multi-line joining**: Read all lines, then merge continuation lines (lines NOT matching `^\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2}`) into the previous logical line with a newline separator.
- **CRITICAL (Pitfall 1 from RESEARCH)**: Do NOT use blind `split(" | ")`. Instead, split from the LEFT: extract timestamp (19 chars), find ` | ` after it for duration, find next ` | ` for status, then find ` | Input: ` marker. For sub-era 1b, find ` | Raw AI: `, ` | Output: `, ` | Pasted: ` markers by searching forward from the Input field end.
- Sub-era detection: lines 1-17 (first 17 logical entries) lack `Raw AI:` -- detect by absence of that marker.
- Parse duration: strip `ms` suffix, convert to int. Preserve negative values as-is (AHK timer bug).
- Parse Raw AI JSON (sub-era 1b): the field uses literal `\\n` (two-char sequences) for newlines. Replace `\\n` with actual `\n` before `json.loads()`. Handle `json.JSONDecodeError` gracefully (set tokens to null).
- Token extraction from Chat Completions response: `usage.prompt_tokens` -> input, `usage.completion_tokens` -> output, `usage.total_tokens` -> total, `usage.prompt_tokens_details.cached_tokens` -> cached (default 0), `usage.completion_tokens_details.reasoning_tokens` -> reasoning (default 0).
- Extract model and model_version from `response["model"]` field in Raw AI JSON.
- Map to JSONL schema: active_app=null, active_exe=null, paste_method=null (not collected), raw_request=null, timings=null, replacements=null, prompt_leak=null, events=null. Set raw_response to the full Raw AI JSON string. Compute text_changed=(input != output), input_chars=len(input), output_chars=len(output). For sub-era 1a (no Raw AI), raw_ai_output=null, raw_response=null, tokens=null. For sub-era 1b, raw_ai_output = output_text (same content in this era). Set error="" for SUCCESS, extract error message for ERROR entries.
- `_converted.source_format`: "pipe-delimited-1a" or "pipe-delimited-1b".

**Format 2 Parser** (detailed multi-section, 19 `spellcheck-detailed*.log` files):
- Read with `encoding="utf-8", errors="replace"`.
- Split file content into entry blocks by the 80-`=` delimiter line (`="*"80`). An entry block spans from one `===...===` header pair to the next.
- **State machine approach**: For each entry block, scan lines and identify sections by their header label. Accumulate indented content (lines starting with `  `) under the current section until the next section header or end-of-block. Section headers are detected by matching known labels: "Status:", "Duration:", "Timing Breakdown:", "Input Text:", "Output Text:", "AI Output (before post-processing):", "Post-processing:", "Prompt-leak safeguard:", "Pasted Text:", "API Response:", "Events:".
- **CRITICAL (Pitfall 3)**: Parse sections by label, NOT by position. Section order varies across eras (2a/2b have API Response before Pasted Text; 2c/2d have it after).
- Parse header line: extract timestamp from `RUN: YYYY-MM-DD HH:MM:SS` or `[ERROR] RUN: YYYY-MM-DD HH:MM:SS`.
- Parse Status: strip value from `Status: {value}`.
- Parse Duration: strip `ms` suffix from `Duration: {N}ms`, convert to int.
- Parse Timing Breakdown (eras 2b+): for each sub-line `  {label}: {N}ms`, map label to JSONL field using the mapping in the interfaces block above. Return null for entire timings object if section absent (era 2a).
- Parse Input Text / Output Text / AI Output / Pasted Text: join indented content lines (strip 2-space prefix), join with newline.
- Parse API Response: accumulate all indented JSON lines, strip 2-space prefix, join, `json.loads()`. Store the joined string as raw_response. Extract tokens using the appropriate field names based on `"object"` value: `"chat.completion"` uses `prompt_tokens`/`completion_tokens`, `"response"` uses `input_tokens`/`output_tokens`. Extract model and model_version from `response["model"]`. Handle error responses (check for top-level `"error"` key) by setting tokens=null.
- Parse Events: each indented line is one event string. Return as list. Return null if section absent (eras 2a/2b based on actual file inspection -- Events section not present in 2a files).
- Parse Post-processing (eras 2c+): parse `no replacements matched` -> {count:0, applied:[], urls_protected:0}. Parse `N replacement(s) applied` -> {count:N, applied:[list from sub-lines], urls_protected:0}. URLs-protected count is not logged in old format, default to 0. Return null if section absent.
- Parse Prompt-leak safeguard (era 2d only): parse `no leak pattern matched` -> {triggered:false, occurrences:0, text_input_removed:false, removed_chars:0, before_length:null, after_length:null}. If triggered, parse details. Return null if section absent.
- Determine output_text: if "AI Output (before post-processing)" exists, use it as raw_ai_output and use "Pasted Text" as output_text. If only "Output Text" exists, use it as both raw_ai_output and output_text.
- Compute text_changed, input_chars, output_chars.
- Set active_app=null, active_exe=null, raw_request=null (not collected in old formats).
- Infer paste_method from events if present: search for "SendText" in events -> "sendtext", search for "Clipboard paste" -> "clipboard", else null.
- `_converted.source_format`: detect era from which sections are present: has Prompt-leak safeguard section -> "detailed-2d", has Post-processing section but no Prompt-leak -> "detailed-2c", has Timing Breakdown but no Post-processing -> "detailed-2b", else -> "detailed-2a".
- `_converted.source_file`: the filename (e.g., "spellcheck-detailed-2025-11-30-125348.log").

**Deduplication (Oct 14 overlap)**:
- Process Format 2 files FIRST (sorted by filename).
- Build a dedup set of `(timestamp_str, input_text[:100])` tuples from all Format 2 entries with dates on 2025-10-14.
- When processing Format 1, for entries dated 2025-10-14, check if `(timestamp, input[:100])` exists in the dedup set. If yes, skip the Format 1 entry (prefer the richer detailed-format version).
- Track and report the dedup skip count.

**Output routing**:
- Collect ALL converted entries into a list, sort by timestamp string (lexicographic sort works for YYYY-MM-DD HH:MM:SS format).
- For each entry, compute week start/end from its timestamp, serialize to JSON (ensure_ascii=False for Unicode preservation, separators=(",", ":") for compact output matching AHK logger style -- actually check existing JSONL format first: existing entries use no-space separators like `{"timestamp":"..."`), append newline.
- Use resolve_weekly_log_path() to route each entry-line to the correct weekly file, respecting 5 MiB soft cap. CRITICAL: never split an entry across files.
- Write entries by opening files in append mode with `encoding="utf-8", newline="\n"`.

**Post-conversion cleanup**:
- Create `logs/archive/` directory if it doesn't exist.
- Move ALL `.log` files from `logs/` to `logs/archive/` (both `OLD - spellcheck.log` and all `spellcheck-detailed*.log` files).

**Error handling**: Wrap each entry's parsing in try/except. On error, log the file, line/entry number, and error message to stderr. Continue processing. Track error count. If errors exceed 50, abort with a summary.
