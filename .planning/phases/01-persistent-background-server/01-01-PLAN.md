**Server structure (single file, all components below):**

**1. Configuration constants (top of file):**
- `LISTEN_HOST = "127.0.0.1"` (per D-03, loopback only)
- `LISTEN_PORT = 48080` (per D-03, fixed port)
- `OPENAI_BASE_URL = "https://api.openai.com"` (base URL for proxied requests)
- `KEEPALIVE_EXPIRY = 120.0` (per D-07, 2 minutes)
- `KEEPALIVE_PING_INTERVAL = 45` (per D-08, seconds between keep-alive pings)
- `PID_FILE` = `os.path.join(os.path.dirname(os.path.abspath(__file__)), "logs", "server.pid")`
- `LOG_FILE` = `os.path.join(os.path.dirname(os.path.abspath(__file__)), "logs", "server.log")`

**2. Logging setup (immediately after constants, per D-11):**
- `os.makedirs(os.path.dirname(LOG_FILE), exist_ok=True)`
- `logging.basicConfig(filename=LOG_FILE, level=logging.INFO, format="%(asctime)s %(levelname)s %(message)s")`
- `log = logging.getLogger("spellcheck-server")`
- This is critical because `pythonw.exe` swallows all stdout/stderr (Research Pitfall 5).

**3. Global state:**
- `http_client: httpx.AsyncClient | None = None`
- `keepalive_task: asyncio.Task | None = None`

**4. PID file management:**
- `write_pid_file()`: Write `os.getpid()` to `PID_FILE`. If file exists, read existing PID, check if process is alive via `os.kill(pid, 0)` (catches `OSError`). If alive, log warning and exit (per D-05, prevent duplicates). If stale, overwrite.
- `remove_pid_file()`: Delete `PID_FILE` with `try/except OSError: pass`.

**5. Startup lifecycle (`async def startup()`):**
- Call `write_pid_file()` -- exit if duplicate detected.
- Create global `http_client = httpx.AsyncClient(http2=True, limits=httpx.Limits(max_keepalive_connections=5, max_connections=10, keepalive_expiry=KEEPALIVE_EXPIRY), timeout=httpx.Timeout(connect=5.0, read=35.0, write=5.0, pool=5.0))`.
- Start keep-alive background task: `keepalive_task = asyncio.create_task(keepalive_ping_loop())`.
- Log: `"Server started on {LISTEN_HOST}:{LISTEN_PORT} (PID {os.getpid()})"`.

**6. Shutdown lifecycle (`async def shutdown()`):**
- Cancel `keepalive_task` if running.
- Close `http_client` with `await http_client.aclose()`.
- Call `remove_pid_file()`.
- Log: `"Server shut down"`.

**7. Keep-alive ping loop (`async def keepalive_ping_loop()`, per D-08):**
- `while True: await asyncio.sleep(KEEPALIVE_PING_INTERVAL)` then `await http_client.get(OPENAI_BASE_URL + "/v1/models", headers={"authorization": auth_header})` where `auth_header` is read from `os.environ.get("OPENAI_API_KEY", "")`.
- Wrap the GET in try/except; on failure log warning but do NOT crash.
- Use the API key from env var to authenticate the ping (unauthenticated requests get 401 and wouldn't exercise the TLS connection properly).
- Format auth header as `"Bearer " + api_key`.

**8. Proxy endpoint (`async def proxy_request(request: Request) -> Response`, per D-01):**
- Read body: `body = await request.body()`
- Forward Authorization header transparently: `auth = request.headers.get("authorization", "")`
- Reconstruct the target URL: the request path from AHK will be `/v1/responses`, so forward to `OPENAI_BASE_URL + request.url.path` (this makes the proxy path-transparent).
- Measure timing: `start = time.perf_counter()` before the httpx call.
- Make the request: `resp = await http_client.post(target_url, content=body, headers={"content-type": "application/json; charset=utf-8", "authorization": auth})`
- Calculate `proxy_ms = round((time.perf_counter() - start) * 1000, 1)`.
- Detect connection reuse: Check `resp.extensions` for HTTP version info. Use `resp.http_version` if available. For connection reuse detection, check `resp.extensions.get("network_stream")` -- if the stream object's underlying connection was reused, `resp.extensions` may contain reuse info. Simpler approach: track whether this is the first request or a subsequent one via a global counter. Set `connection_reused = "true"` if counter > 1, `"false"` if counter == 1. Increment counter on each successful response.
- Build response headers (per D-10, only real measurements — no placeholders):
  ```python
  response_headers = {
      "content-type": resp.headers.get("content-type", "application/json"),
      "x-proxy-ms": str(proxy_ms),
      "x-connection-reused": connection_reused,
  }
  ```
- TLS/DNS/TCP breakdown headers deferred to Phase 4 when httpx transport-level instrumentation is added.
- Return `Response(content=resp.content, status_code=resp.status_code, headers=response_headers)`.
- Error handling: catch `httpx.ConnectError` -> return 502 JSON `{"error": "upstream_connect_failed", "detail": str(e)}` with same timing headers. Catch `httpx.TimeoutException` -> return 504 JSON `{"error": "upstream_timeout", "detail": str(e)}` with timing headers. Log both errors.

**9. Health endpoint (`async def health(request: Request) -> Response`):**
- Return `JSONResponse({"status": "ok"})`.

**10. Starlette app assembly:**
```python
app = Starlette(
    routes=[
        Route("/v1/responses", proxy_request, methods=["POST"]),
        Route("/health", health, methods=["GET"]),
    ],
    on_startup=[startup],
    on_shutdown=[shutdown],
)
```

**11. Main block:**
```python
if __name__ == "__main__":
    try:
        uvicorn.run(app, host=LISTEN_HOST, port=LISTEN_PORT, log_level="warning")
    except Exception as e:
        log.critical("Server failed to start: %s", e)
        remove_pid_file()
```

- Do NOT hardcode `pythonw.exe` path in the server itself (that's AHK's job in Plan 02).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AHK -> localhost proxy | Trusted: loopback only, same machine, same user |
| Proxy -> api.openai.com | Untrusted network: TLS-encrypted, auth token forwarded |
| Environment -> server | API key read from environment variable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | proxy endpoint | mitigate | Bind to 127.0.0.1 only (not 0.0.0.0). Only localhost processes can connect. No auth on local endpoint because only local AHK connects. |
| T-01-02 | Information Disclosure | API key in environment | mitigate | Read from OPENAI_API_KEY env var (per SEC-01/D-09). Never log the API key. Server forwards Authorization header transparently -- does not store it. Keep-alive ping reads key from env var directly. |
| T-01-03 | Denial of Service | PID file race | accept | Single-user personal tool. PID check at startup is sufficient. No multi-user contention scenario. |
| T-01-04 | Information Disclosure | server.log | accept | Log file contains only operational messages (startup, shutdown, errors). No request/response bodies logged. No API keys logged. File permissions inherit user's home directory. |
| T-01-05 | Tampering | proxy request/response | accept | Localhost loopback. Attacker with local access already has full machine access. No additional protection needed beyond OS-level. |
| T-01-06 | Elevation of Privilege | server process | accept | Runs as same user as AHK script. No elevated privileges. pythonw.exe runs in user context. |
</threat_model>
