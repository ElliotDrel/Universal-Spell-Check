<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment -> AHK script | API key read from OPENAI_API_KEY env var |
| AHK -> localhost proxy | Plain HTTP on loopback, same user |
| User input -> clipboard -> API | Untrusted text sent to OpenAI for correction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-07 | Information Disclosure | API key in AHK | mitigate | Remove hardcoded `sk-proj-...` key from source code. Read from `OPENAI_API_KEY` env var via `EnvGet()`. Key never logged -- only passed in Authorization header. |
| T-01-08 | Denial of Service | proxy health check | accept | 300ms timeout on health check. If proxy is down, shows error tooltip immediately. No retry loop in hot path. |
| T-01-09 | Spoofing | proxy response | accept | Localhost only. Attacker with local access already owns the machine. No additional verification of proxy responses needed. |
| T-01-10 | Repudiation | logging | mitigate | All proxy interactions logged to JSONL with timing headers. `proxy_ms` and `connection_reused` provide audit trail of server behavior. |
</threat_model>
